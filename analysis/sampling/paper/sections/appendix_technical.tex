\subsection{Synthetic Network Generation}

Synthetic networks are generated using tile-based templates. Each template defines a connectivity pattern within a single tile; tiles are then arrayed to create networks of the desired extent.

\begin{description}
\item[Trellis] A regular grid with diagonal connections, representing highly connected urban cores. Average degree $\approx 4$.

\item[Tree] A hierarchical branching structure where each node connects to at most 3 others. Represents suburban cul-de-sac development. Average degree $\approx 2.3$.

\item[Linear] A constrained grid with fewer cross-connections than trellis. Intermediate connectivity. Average degree $\approx 3$.
\end{description}

Networks are tiled 24$\times$24 to achieve approximately 12km extent, then the largest connected component is extracted. Node positions are perturbed slightly to avoid perfectly regular spacing.

\subsection{GLA Network Preprocessing}

The Greater London Authority road network is sourced from Ordnance Survey Open Roads data (GeoPackage format).

Preprocessing steps:
\begin{enumerate}
\item Remove invalid and empty geometries
\item Explode multi-part geometries
\item Build NetworkX graph from edges
\item Remove filler nodes (degree-2 nodes that don't represent junctions)
\item Remove dangling nodes (dead-end segments $< 20$m)
\item Consolidate nearby nodes (threshold 1m)
\item Extract largest connected component
\end{enumerate}

Final network: $\approx$ \glaNnodes{} nodes.

\subsection{Correctness Verification Against NetworkX}
\label{sec:appendix_networkx}

To ensure the correctness of our Rust-based centrality implementation in \cityseer{}, we conducted systematic verification against the widely-used NetworkX library \citep{Hagberg2008}. For each synthetic network topology and distance threshold combination, we computed both harmonic closeness and betweenness centrality using our Rust implementation (via \cityseer{} Python bindings) and NetworkX's reference functions.

For each node $v$ in the test networks, we computed the absolute difference $\Delta_v = |C_{\text{cityseer}}(v) - C_{\text{NetworkX}}(v)|$. The maximum difference across all nodes and all test configurations was $\max_v \Delta_v < 10^{-6}$, attributable to floating-point arithmetic precision.

Verification was performed across all three synthetic topologies, distance thresholds from 200m to 3200m, both shortest-path and angular distance heuristics, and both harmonic closeness and betweenness centrality.

\subsection{H\'{a}jek Estimator}
\label{sec:appendix_hajek}

When estimating population totals from samples, two classical approaches exist: the Horvitz-Thompson (H-T) estimator and the H\'{a}jek estimator. We selected the H\'{a}jek estimator based on theoretical considerations.

The Horvitz-Thompson estimator \citep{Horvitz1952} estimates a population total as $\hat{T}_{HT} = \sum_{i \in S} y_i / \pi_i$, where $S$ is the sample and $\pi_i$ is the inclusion probability. The H\'{a}jek estimator \citep{Hajek1971} instead uses a ratio form:
\begin{equation}
    \hat{T}_{H} = N \cdot \frac{\sum_{i \in S} y_i / \pi_i}{\sum_{i \in S} 1 / \pi_i}
\end{equation}
where $N$ is the known population size. For simple random sampling with equal inclusion probabilities, the H\'{a}jek estimator has variance $\text{Var}(\hat{T}_{H}) \approx \text{Var}(\hat{T}_{HT}) \cdot (1 - n/N)$. Since the population size (total number of network nodes) is known, the H\'{a}jek estimator is the natural choice. In our implementation, centrality contributions are accumulated unscaled during traversal, then multiplied by $N/n$ (total nodes divided by sampled nodes) after all sources have been processed.

\subsection{Implementation in \cityseer{}}
\label{sec:appendix_cityseer_impl}

The sampling model is implemented in the \cityseer{} Python library (version 4.23+). The adaptive function automatically probes reachability, computes sampling probabilities from the fitted model, and scales results:

\begin{verbatim}
from cityseer.metrics import networks

gdf = networks.node_centrality_shortest_adaptive(
    network_structure,
    nodes_gdf,
    distances=[500, 1000, 2000, 5000, 10000],
    compute_closeness=True,
    compute_betweenness=True,
    target_rho=0.95
)
\end{verbatim}

For manual control, a fixed sampling probability can be specified via the \texttt{sample\_probability} parameter of \texttt{node\_centrality\_shortest}. Helper functions for custom workflows (\texttt{compute\_required\_p}, \texttt{probe\_reachability}, \texttt{spatial\_sample}) are available in the \texttt{cityseer.config} module; see the library documentation for details.

\subsection{Localised Eppstein--Wang Bound: Detailed Validation}
\label{sec:appendix_ew}

We evaluate the localised Hoeffding-based error guarantee at the rank model's recommended sample sizes across all 440 synthetic configurations. At each configuration, the observed normalised error $\varepsilon_{\text{obs}}$ (maximum absolute error divided by the theoretical maximum) is compared against the predicted bound $\varepsilon_{\text{pred}} = \sqrt{\log(2r/\delta) / (2 \cdot \effn)}$ with $\delta = 0.1$.

Overall, 412 of 440 configurations (93.6\%) satisfy the bound, exceeding the nominal 90\% guarantee. Harmonic closeness achieves 100\% success (220/220), while betweenness achieves 87.3\% (192/220). The betweenness shortfall is not statistically significant: under the null hypothesis that the true success rate is 90\%, the observed count falls within 1.4 standard deviations of the expectation ($n = 220$, $\sigma = 4.4$). Violations concentrate at low reach ($r < 100$, corresponding to analysis distances $\leq 500$\,m), where $\effn < 10$ and the Hoeffding inequality is in its degenerate small-sample regime. Conditional on reach $\geq 100$, the success rate rises to 99.7\% overall (307/308) and 99.4\% for betweenness (153/154).

The bound is conservative: median predicted-to-observed epsilon ratios are $3.8\times$ for betweenness and $260\times$ for harmonic closeness (Figure~\ref{fig:ew_synthetic}A). This gap reflects two factors: the Hoeffding inequality assumes worst-case bounded summands, whereas per-source contributions on planar networks have substantially lower variance; and worst-case uniform bounds do not exploit the spatial correlation of errors across neighbouring nodes.
